# rednoise (Rust bindings)

Safe Rust bindings for the RedNoise CPU software renderer. They wrap the stable
C ABI declared in `include/rednoise/rednoise.h`.

The crate builds the C++ engine **from source**: there is no prebuilt library to
install and no network access needed at build time. Adding the crate and
building it just works.

```sh
cargo add rednoise
```

## Prerequisites

A C++23 compiler. The `cc` crate locates it automatically:

- Linux: gcc 13+ or clang 16+ (`g++` / `clang++` on `PATH`)
- macOS: a recent Xcode / Command Line Tools clang
- Windows: Visual Studio 2022 (MSVC) with the C++ workload

Nothing else. OpenMP is optional: the engine builds and runs single-threaded
without it (the `#pragma omp` hints are ignored), so no OpenMP runtime is
required.

## Building the crate

```sh
cargo build
```

The build script (`build.rs`) compiles the vendored C++ engine with the `cc`
crate at C++23 into a static library and links it in, along with the platform
C++ runtime (`stdc++` on Linux, `c++` on macOS, the toolchain default on
Windows). No `REDNOISE_LIB_DIR` and no CMake step are needed.

Optional fast path: if you already have a prebuilt `rednoise` dynamic library,
set `REDNOISE_LIB_DIR` to the directory containing it and the build script links
that instead of compiling from source. This is off by default.

## Usage

```rust
use rednoise::{Scene, Mode};

fn main() -> Result<(), rednoise::Error> {
    let scene = Scene::load_obj("assets/cornell-box.obj", 0.35)?;
    println!("triangles: {}", scene.triangle_count());

    let image = scene.render(Mode::Pathtraced, 400, 300, 4.0, 64)?;
    image.save_png("cornell.png")?;

    println!("rednoise {}", rednoise::version());
    Ok(())
}
```

## Example

Run the bundled example from the repository root (so the asset path resolves):

```sh
cargo run --manifest-path bindings/rust/Cargo.toml --example render
```

The engine is statically linked, so there is no shared library to locate at
runtime.

## Vendored sources

crates.io only packages files inside the crate directory, so the engine sources
(which live at the repo root) are copied under `vendor/` before publishing.
`vendor/` is git-ignored to keep the main repo lean and is regenerated by
`vendor.sh`:

```sh
bindings/rust/vendor.sh
```

PowerShell equivalent (run from the repo root):

```powershell
$r = "bindings/rust/vendor"
Remove-Item -Recurse -Force $r -ErrorAction SilentlyContinue
New-Item -ItemType Directory -Force $r | Out-Null
foreach ($d in "framework","src","include","third_party") {
    Copy-Item -Recurse -Force $d (Join-Path $r $d)
}
```

`Cargo.toml`'s `include` list packages `vendor/` even though it is git-ignored,
and the `publish-rust` GitHub Actions workflow runs `vendor.sh` before
`cargo publish`, so the published crate is fully self-contained. When building
from a fresh source checkout, run `vendor.sh` once first.

## API

- `Scene::load_obj(path, scale) -> Result<Scene, Error>`
- `Scene::triangle_count() -> u32`
- `Scene::render(mode, width, height, cam_z, samples) -> Result<Image, Error>`
- `Image { width, height, rgba }` with `Image::save_png(path)`
- `Mode`: `Wireframe`, `Rasterised`, `Raytraced`, `Pathtraced`
- `version() -> String`
